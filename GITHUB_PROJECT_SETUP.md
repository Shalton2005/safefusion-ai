# SafeFusion AI Development Board

## Project Summary

**Project Name:** SafeFusion AI Development Board  
**Project Description:** Development tracker for SafeFusion AI, an AI-powered Industrial Safety Intelligence Platform built for ET AI Hackathon 2.0.

This board is intended to manage the complete software development lifecycle across planning, architecture, backend, frontend, AI modules, testing, documentation, and final submission.

The objective is to build a production-inspired prototype that integrates IoT sensors, SCADA systems, CCTV analytics, permit-to-work data, maintenance logs, worker tracking, and regulatory knowledge into a unified AI-powered decision intelligence platform for Zero-Harm Operations.

---

## Project Goal

Develop a robust AI-powered Industrial Safety Intelligence Platform capable of proactively identifying compound industrial risks through data fusion, agentic AI, computer vision, knowledge graphs, geospatial analytics, and retrieval-augmented generation.

The project follows an iterative development workflow with continuous integration, testing, documentation, and demo preparation over a 20-day hackathon timeline.

## Kanban Columns

Create the GitHub Project board with these columns in this order:

| Column | Description |
| --- | --- |
| Backlog | Ideas and future tasks that are identified but not yet scheduled for development. |
| To Do | Tasks that are planned and ready to begin. |
| In Progress | Tasks currently being developed by team members. |
| Testing | Completed development tasks undergoing testing, debugging, validation, and integration. |
| Done | Successfully completed, tested, and merged tasks. |

Suggested display names:

- `📌 Backlog`
- `📝 To Do`
- `🚀 In Progress`
- `🧪 Testing`
- `✅ Done`

## Suggested Usage

- Put unscheduled ideas in `📌 Backlog`.
- Move ready tasks into `📝 To Do`.
- Assign active work to one owner and move it into `🚀 In Progress`.
- Move completed implementation tasks into `🧪 Testing` for validation and integration.
- Move verified items into `✅ Done` only after testing is complete.

## Recommendation

Because the team has only two members, keep the GitHub Project simple.

- Use a lightweight Kanban board.
- Keep issues clearly titled and assigned.
- Rely on labels and milestones instead of heavy automation.
- Review and update the board daily during the 20-day sprint.

This is enough to keep the workflow visible without creating unnecessary project-management overhead.
